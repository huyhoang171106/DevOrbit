# Execution Blueprint — Plan 1.1: Fix Mobile Data Integrity (Data Sync)

## 1. Mission & Scope

### 1.1 Mission Statement
Eliminate the deserialization mismatch between the Kotlin Android data models and the JSON payloads served by the Spring Boot API, ensuring that TechStack data is correctly parsed and rendered on the Repository Detail screen without crashes or silent data loss.

### 1.2 In-Scope
- `TechStack.kt` — Kotlin data class alignment with `TechStackResponse.java` JSON shape
- `RepoSummary.kt` — `techStacks` field deserialization (null safety, default initialization)
- `RepoDetailScreen.kt` — UI consumption of `TechStack.name` (read-only verification; no UI logic changes needed)
- Cache layer in `DevOrbitRepository.kt` — GSON round-trip serialization/deserialization of cached repo data (must survive cache save/load cycle)

### 1.3 Out-of-Scope
- Adding new fields to `TechStackResponse` on the backend
- Changing the API contract (no backend modifications)
- Backend test coverage (Phase 3 concern)
- Mobile UI styling, theming, or layout changes
- Bookmark, authentication, or course-related models

### 1.4 Success Criteria (SMART)

| Criterion | Measure | Verification Method |
|-----------|---------|---------------------|
| TechStack objects deserialize from API JSON without GSON errors | Zero `JsonSyntaxException` / `JsonIOException` in logcat | Run against live API with a course containing repos with tech stacks |
| `RepoSummary.techStacks` is non-null for repos that have tech stacks | `repo.techStacks` returns a `List<TechStack>` with correct size | Log `repo.techStacks.size` on RepoDetailScreen load |
| Tech stack names render correctly in the UI | `it.name` accesses on each `TechStack` produce the expected string | Visual inspection: "Java, Spring Boot" displayed at line `RepoDetailScreen.kt:66` |
| Cached data round-trips correctly via GSON | Cache save (`gson.toJson`) followed by load (`gson.fromJson`) preserves `techStacks` faithfully | Unit test covering `TechStack` + `RepoSummary` serialization round-trip |

### 3.3 Risk Register

| ID | Risk | Likelihood | Impact | Mitigation | Contingency |
|----|------|------------|--------|------------|-------------|
| R-01 | GSON uses different default field naming than Jackson (e.g., `primaryLanguage` vs `primary_language`) | Low | High | Confirm `GsonConverterFactory.create()` default is field-name exact match; do not enable `LOWER_CASE_WITH_UNDERSCORES` | Add `@SerializedName` annotations if mismatch found |
| R-02 | Cache stores stale model version — old JSON with `techStack` (singular) instead of `techStacks` | Low | Medium | Clear app cache / reinstall during testing | Add cache-busting version key or migration logic |
| R-03 | `TechStack` JSON includes backend entity fields like `id` that Kotlin model doesn't declare | Medium | Low | GSON ignores unknown fields by default; verify `setLenient()` not needed | Add `@SerializedName` on extra fields for forward compatibility |
| R-04 | Android emulator cannot reach `10.0.2.2:8080` | Low | High | Verify backend is running in Docker; check `docker ps` and port mapping | Use `adb reverse tcp:8080 tcp:8080` as fallback |
| R-05 | Backend returns `techStacks: null` instead of `techStacks: []` | Low | Medium | `RepoSummary.techStacks` already has `= emptyList()` default | Ensure `RepoSummaryResponse` uses `@JsonInclude(NON_NULL)` or empty list |
