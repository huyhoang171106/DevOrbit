# Ke Hoach Trien Khai Phase-2 DevOrbit

## 1. Muc Tieu Phase-2

Phase-2 mo rong DevOrbit tu mot nen tang curated Legacy Repo Finder thanh mot he sinh thai hoc tap co ca nhan hoa, ghi chu, roadmap, knowledge graph va tin hieu cong dong, trong khi van giu chat luong curated cua MVP.

Muc tieu san pham:

- Cho phep sinh vien dang nhap, luu tien do hoc tap, ghi chu va bookmark repo/mon hoc.
- Cung cap learning roadmap theo mon hoc, tech stack hoac muc tieu nghe nghiep.
- Truc quan hoa quan he giua mon hoc, prerequisite, repo va tech stack bang knowledge graph.
- Them split-view note taking de hoc sinh vien vua xem tai lieu/repo vua ghi chu.
- Them rating/voting/feedback co kiem soat chat luong cho repo va roadmap.
- Them scheduled background scanning de cap nhat metadata GitHub va de xuat candidate moi.
- Nang cap admin thanh CMS nhe voi role hierarchy co gioi han.

Muc tieu ky thuat:

- Giu backwards compatibility voi public/admin API cua MVP.
- Uu tien migration an toan, feature flags va rollout tung phan.
- Khong lam loang core workflow: public approved repos van la nguon du lieu tin cay.
- Tang test coverage cho cac flow co du lieu nguoi dung va permission.

## 2. Gia Dinh Dau Vao

Do chua co thong tin chi tiet ve team, deadline va ha tang production, ke hoach nay dua tren cac gia dinh hop ly sau:

- MVP da hoan thanh va co cac module chinh: `devorbit-api`, `devorbit-web`, `devorbit-mobile`, PostgreSQL, JWT admin auth, GitHub scan/approve workflow.
- Team trien khai gom 1 backend engineer, 1 frontend web engineer, 1 mobile engineer, 1 UI/UX/product owner ban thoi gian va 1 QA ban thoi gian.
- Phase-2 duoc thuc hien trong 12 tuan, chia thanh 6 milestones moi milestone 2 tuan.
- Student account system co the dung email/password truoc, SSO/UIT domain verification de sau neu chua co access vao he thong dinh danh cua truong.
- PostgreSQL tiep tuc la database chinh. Neu can graph query nang cao thi van bat dau bang relational model truoc, chua dua Neo4j vao ngay.
- Scheduled jobs chay trong backend Spring Boot truoc. Neu tai cao moi tach worker rieng.
- Mobile Phase-2 uu tien student account, roadmap read-only, bookmark, note co ban. Admin va CMS khong len mobile.

## 3. Pham Vi Cong Viec

### 3.1 In Scope

- Student account system:
  - Dang ky, dang nhap, refresh/session handling.
  - Profile co ban.
  - Saved courses, bookmarks, learning progress.
  - Password reset neu co email service; neu chua co thi de trong milestone hardening.

- Learning roadmap:
  - Admin/editor tao roadmap.
  - Roadmap gom modules, steps, linked courses, repos, tech stacks.
  - Sinh vien xem roadmap va mark progress.
  - Roadmap theo subject/tech stack/career goal.

- Knowledge graph:
  - Data model cho nodes va edges.
  - Quan he prerequisite, related-to, uses-tech, references-repo.
  - UI graph explorer tren web.
  - Admin/editor quan ly relationship co ban.

- Split-view notes:
  - Ghi chu ca nhan gan voi course, repo, roadmap step.
  - Split-view UI tren web.
  - CRUD notes, autosave co debounce.
  - Mobile notes co ban neu con du thoi gian.

- Social rating/voting:
  - Upvote/downvote hoac rating cho repo/roadmap.
  - Feedback/comment ngan co moderation.
  - Anti-abuse co ban: mot user mot vote, report, hide.

- Scheduled scanning:
  - Scheduled GitHub metadata refresh.
  - Scheduled candidate discovery theo saved scan queries.
  - Admin dashboard cho job status va errors.

- CMS/role hierarchy:
  - Roles: `SUPER_ADMIN`, `EDITOR`, `MODERATOR`, `VIEWER`.
  - Permission guard cho admin endpoints.
  - CMS nhe cho landing content, announcements, featured roadmaps/repos.

- Observability va quality:
  - Audit logs cho admin actions.
  - Metrics cho adoption, roadmap completion, scan quality.
  - Test suite cho core Phase-2 flows.

### 3.2 Out Of Scope Cho Phase-2

- Realtime collaborative notes.
- Full LMS thay the Moodle/Canvas.
- AI tutor/chatbot production-grade.
- Native graph database migration bat buoc.
- Complex organization management da cap.
- Payment, certificate issuance, plagiarism checking.

## 5. Cac Dau Viec Chinh Theo Hang Muc

### 5.1 Foundation: Auth, Permissions, Audit

Backend:

- Thiet ke schema `student_users`, `student_profiles`.
- Them password hashing, JWT/refresh token strategy cho student.
- Tach auth filter de ho tro ca admin va student.
- Them role hierarchy cho admin.
- Them authorization annotations/service guards cho admin endpoints.
- Them `audit_logs` cho admin changes va moderation actions.

Web:

- Trang student login/register.
- Auth provider/client token handling.
- Protected student routes.
- Role-aware admin layout.

Mobile:

- Login screen.
- Secure token storage.
- Authenticated API client.

Tests:

- Login/register tests.
- Permission tests cho admin roles.
- Token expiry/invalid token tests.

### 5.2 Learning Roadmap

Backend:

- Schema roadmap/modules/steps/resources.
- Admin CRUD APIs cho roadmap.
- Public/student APIs de browse roadmap.
- Student progress APIs: start, complete step, reset.
- Link resources den course, repo, tech stack va external URL.

Web:

- Roadmap list page.
- Roadmap detail with module/step timeline.
- Progress UI.
- Admin roadmap editor.

Mobile:

- Roadmap list/detail.
- Mark step complete.

Tests:

- Roadmap CRUD.
- Progress calculation.
- Visibility rules: draft/published/archived.

### 5.3 Knowledge Graph

Backend:

- Schema generic graph nodes/edges.
- Node types: `COURSE`, `REPO`, `TECH_STACK`, `ROADMAP`, `ROADMAP_STEP`, `CONCEPT`.
- Edge types: `PREREQUISITE_OF`, `RELATED_TO`, `USES_TECH`, `REFERENCES`, `PART_OF`, `NEXT_STEP`.
- Graph query APIs: neighbors, subgraph by course/tech stack, shortest relation depth limited.
- Admin APIs tao/sua/xoa edges va concept nodes.

Web:

- Graph explorer using React graph library.
- Course detail graph section.
- Admin relationship manager.

Tests:

- Edge validation.
- Cycle handling for prerequisites.
- Query performance with seeded graph.

### 5.4 Split-View Notes And Bookmarks

Backend:

- `student_notes` CRUD.
- Note scopes: course, repo, roadmap step, freeform.
- Bookmark APIs cho course/repo/roadmap/step.
- Autosave endpoint with optimistic concurrency field `updatedAt` or `version`.

Web:

- Split-view layout on repo/course/roadmap detail.
- Notes editor with autosave status.
- My Notes page.
- Bookmark button components.

Mobile:

- Bookmark support.
- Notes list/detail basic editor if feasible.

Tests:

- Student can only access own notes.
- Autosave conflict behavior.
- Bookmark idempotency.

### 5.5 Social Rating, Voting, Feedback

Backend:

- Voting APIs for repos and roadmap.
- Aggregate fields or materialized counters: score, rating count, average rating.
- Feedback/comment APIs.
- Moderation status: `VISIBLE`, `HIDDEN`, `REPORTED`, `REMOVED`.
- Report content API.

Web:

- Vote/rating controls on repo cards/details.
- Feedback form and list.
- Admin moderation queue.

Mobile:

- Basic vote/rating controls.

Tests:

- One vote per user per target.
- Aggregation correctness.
- Moderation hides reported/removed content from public views.

### 5.6 Scheduled Background Scanning

Backend:

- `scheduled_scan_queries` linked to course and admin owner.
- `scan_jobs` for execution history, status, error message, duration, candidate count.
- Spring `@Scheduled` jobs for metadata refresh and candidate scan.
- Rate limit handling and backoff.
- Deduplication rules respecting rejected candidates.

Admin Web:

- Schedule scan query management.
- Job dashboard with status/error.
- Manual rerun button.

Tests:

- Job creates candidates without duplicates.
- Failed GitHub request does not publish data.
- Rate limit produces clear job failure reason.

### 5.7 CMS And Admin Role Hierarchy

Backend:

- Roles and permissions seed data.
- CMS page/block schema.
- CMS APIs for announcements, featured roadmaps, featured repos.
- Audit logs for CMS edits.

Web:

- CMS editor for simple structured blocks.
- Public rendering for announcements/featured sections.
- Role restricted navigation/actions.

Tests:

- Editor can manage content but not users.
- Moderator can hide feedback but not edit roadmaps.
- Viewer is read-only.

### 5.8 Observability, Analytics, Quality

Backend:

- Add event tracking table or lightweight analytics logging.
- Track key events: signup, login, roadmap_started, step_completed, repo_opened, vote_submitted, note_created.
- Admin metrics endpoints.
- Add health endpoints for scheduled jobs.

Web/Admin:

- Phase-2 dashboard with adoption and content quality metrics.

Tests/Quality:

- Regression suite for MVP flows.
- API contract checks for web/mobile shared DTOs.
- Load smoke test for graph and roadmap APIs.

## 11. KPI/Metrics Danh Gia Hieu Qua

### 11.1 Adoption Metrics

- Student signup count.
- Weekly active students.
- Login success rate.
- % users co it nhat 1 bookmark.
- % users start it nhat 1 roadmap.

Target goi y 4 tuan sau release:

- 100+ student accounts neu rollout trong mot cohort nho.
- 30%+ logged-in users tao bookmark.
- 20%+ logged-in users start roadmap.

### 11.2 Learning Engagement Metrics

- Roadmap starts per roadmap.
- Roadmap step completion rate.
- Average completed steps per active student.
- Notes created per active student.
- Repo opens from roadmap steps.

Target goi y:

- 25% roadmap starters complete it nhat 3 steps.
- 15% active students tao it nhat 1 note.
- 2+ repo opens per roadmap starter.

### 11.3 Content Quality Metrics

- Repo average rating.
- Vote count per approved repo.
- Feedback/report ratio.
- Candidate-to-approved conversion rate.
- % repos with updated metadata in last 30 days.

Target goi y:

- 80% approved repos co metadata refreshed trong 30 ngay.
- Reported feedback < 5% total feedback.
- Candidate duplicate rate giam qua moi scan cycle.

### 11.4 Admin Productivity Metrics

- Time from scheduled scan to candidate review.
- Number of candidates reviewed per admin session.
- Roadmaps published per week.
- Moderation average resolution time.
- Failed job resolution time.

Target goi y:

- 80% reported content duoc xu ly trong 72 gio.
- 80% failed scan jobs co error message actionable.
- Admin co the publish roadmap co ban trong < 30 phut voi data san co.

### 11.5 Technical Health Metrics

- API p95 latency cho roadmap detail < 500ms tren staging dataset.
- API p95 latency cho graph query < 800ms voi depth limit.
- Scheduled job success rate > 95% tru cac loi GitHub quota.
- Backend test pass rate 100% tren CI.
- Error rate tren student APIs < 1%.
- No critical/high security findings open at release.

## 13. Rollout Strategy

- Use feature flags or config toggles per epic where possible.
- Release order:
  - Internal staging with seed users and sample roadmaps.
  - Admin-only beta for roadmap/CMS/job features.
  - Student beta for auth/bookmarks/roadmaps/notes.
  - Enable voting/feedback after moderation workflow is ready.
  - Enable scheduled scanning with conservative interval first.
- Keep read-only fallback: if Phase-2 student auth fails, public MVP browsing remains available.
- Prepare rollback by keeping migrations additive in early milestones and avoiding destructive column changes.
