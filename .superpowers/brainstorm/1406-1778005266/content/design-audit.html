DevOrbit Redesign: Design Audit Results

Key problems found in the current design — pick what to prioritize

Typography

  • Inter everywhere — generic AI/startup default. Sora + DM Sans loaded but unused.
  • No text-wrap: balance on headings (orphaned words)
  • No tabular-nums for data-heavy interfaces

Color & Surfaces

  • Sky-blue gradient hero — generic SaaS/AI aesthetic, doesn't say "academic tool"
  • Flat surfaces — no texture, grain, or depth on backgrounds
  • Untinted shadows
  • Random dark sections (mockup card in hero) breaking light context

Layout

  • 3 equal feature card columns — the most generic AI layout pattern
  • No visual overlap or depth between elements
  • Symmetrical padding everywhere

Interactivity

  • No pressed/active states on buttons
  • Spinner instead of skeleton loaders
  • Sun/moon toggle (cliche)
  • No 404 page, no skip-to-content link

Which direction should we go?

Pick one (or tell me your own idea)

A

Academic / Editorial

Warm, scholarly feel. Serif headings (Sora), muted earth tones, generous whitespace, magazine-style typography. Feels like a university publication.

B

Modern Technical / Developer

Dark-mode capable, code-friendly. DM Sans or Geist for body, monospace for data. Clean, dense, functional. Feels like GitHub + Linear.

C

Dark Space / Cosmic

Deep navy/space backgrounds, accent cyan or amber. Feels like an observatory control panel. Pairs with the "Orbit" name.

D

Clean Minimalist

What exists now but refined. Keep the basic structure, fix the audit issues. Lowest effort, good result.
